# Implementation Plan: Derive quiz test cases from chat export

**Branch**: `002-derive-tests-from-questions` | **Date**: 2026-04-08 | **Spec**: [spec.md](./spec.md)  
**Input**: Feature specification from `/specs/002-derive-tests-from-questions/spec.md`

## Summary

Deliver a **read-only pipeline** that ingests a **Telegram chat export** (zip with `messages.html`) and emits **structured quiz test cases**: player-facing `questionText`, separate **expected answers** when present in spoilers, **traceability** (`sourceMessageId`, `sourceOrder`, optional `Вопрос N` / round hints), plus an **extraction summary** with counts and warnings. Implementation lives in **TypeScript** under `src/chat-export/` (pure parse + extract, **no Cloudflare Worker** dependency). Per **constitution v1.4.0**, the operator/agent entry is an **MCP tool** (`extract_quiz_cases_from_telegram_export`) registered beside existing tools in `mcp/src/index.ts`, with **Zod** schemas and JSON output matching [contracts/quiz-cases-output.schema.json](./contracts/quiz-cases-output.schema.json).

**Primary dependencies** (see [research.md](./research.md)): `node-html-parser`, `fflate`.

## Technical Context

**Language/Version**: TypeScript (ES modules), Node 20 (local + CI)  
**Primary Dependencies**: **node-html-parser**, **fflate**; **@modelcontextprotocol/sdk** + **zod** (MCP tool); **@biomejs/biome** + **vitest** (existing)  
**Storage**: N/A — in-memory parse; optional write of JSON left to operator  
**Testing**: **Vitest** (`tests/`); fixtures under `tests/fixtures/chat-export/` using **realistic HTML slices** from Telegram exports  
**Target Platform**: Node (MCP process + CLI tests); not deployed to Workers  
**Project Type**: Library + **stdio MCP tool** extension (same `mcp/` workspace)  
**Performance Goals**: Process typical channel exports (single large `messages.html`) in seconds on a developer machine; streaming not required for v1  
**Constraints**: Read-only filesystem; log paths + counts, not full message bodies; no bot tokens for this feature  
**Scale/Scope**: One export per invocation; Russian/English text in sources
